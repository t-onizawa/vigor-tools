## Backlogの構成：Tools と Platform

Backlogは2種類の異なるものを扱う。

```
Tools
  ユーザーの個別の迷いを整理する、単体のツール。
  例：生活防衛資金シミュレーター、サブスク年間総額計算

Platform
  ツール横断で機能する、サイト全体の基盤機能。
  例：判断履歴、マイVIGOR、共通検索
```

Toolsは「1本ずつ公開して反応を見る」対象。
Platformは「サイトの前提を変える」対象なので、より慎重に、
小さい単位で検証しながら進める。

# Platform

サイト全体の基盤機能。個別ツールと違い、一度入れると
全ツールの設計に影響するため、最小単位で検証しながら進める。

| 状態 | 機能名 | 説明 |
|---|---|---|
| 検証中 | 判断履歴 | ツールを使った結果を、ユーザーが明示的に残せる仕組み。トレンドやグラフは持たず、ツールごとの「最新の結果」だけを記録する。生活防衛資金シミュレーター・サブスク年間総額計算・続ける理由vs辞めたい理由比較シートの3本で稼働中。 |
| 検証中 | 共通データ（Profile） | 複数ツールで再利用する「入力値」を保存する仕組み。判断履歴（結果のログ）とは別物。サブスク年間総額計算→固定費チェック／生活防衛資金シミュレーターの1本目の連携で稼働中。 |
| 保留 | マイVIGOR | 判断履歴・共通データが自然に育った先に生まれる、状態が見えるホーム画面。両方の価値が検証できるまで着手しない。 |
| 保留 | 共通検索 | ツール数が増えた際の横断検索。目安としてツールが20本を超えたあたりで検討。 |

### 判断履歴 v0.1（検証スコープ）

```
- 対象ツール：生活防衛資金シミュレーター、サブスク年間総額計算、
  続ける理由vs辞めたい理由比較シート（3本で検証中）
- 保存内容：ツールごとの最新の結果1件（時系列やトレンドは持たない）
- 保存先：localStorage（サーバー送信なし、ログイン不要）
- 見せ方：ツール側に「この結果は判断履歴に保存されました。」の一言
  ＋「判断履歴を見る」リンクのみ。専用ホーム画面はまだ作らない。
- 検証すること：履歴が残ること自体が、ユーザーにとって嬉しいか
```

3本での検証が揃った。反応が良ければ、独立準備チェックなど4本目を追加し、
その先で初めて「マイVIGOR」の着手を検討する。

### 共通データ（Profile）v0.1（検証スコープ・2026-07-15新設）

```
- 目的：一度入力した数値を、何度も入力させない
- 対象フィールド：monthlySubscriptionTotal（サブスク月額合計）のみ
- 保存先：localStorage（vigor_profile。判断履歴とは別のキー）
- 保存方式：ユーザーが明示的にボタンを押した時だけ保存する（自動保存はしない）
- 見せ方の原則：数値を「自動入力」せず「参考値として表示」する。
  固定費チェックはサブスク合計を要約表示するが、生活防衛資金シミュレーターの
  「毎月の生活費」フィールドには自動入力しない
  （サブスクは生活費の一部でしかなく、自動入力すると数値が過小になり
  誤解を招くため。Tool Principle 8「前提と責任範囲を明記する」に基づく判断）
- 将来の展開候補：固定費チェック自体に数値入力を持たせるかどうかは
  別途検討（現状はチェックボックスのみで、合計金額を持たない）
```

反応が良ければ、独立準備チェック・退職チェックなど、
固定費データを使う他ツールへの接続を追加する。

**設計上の制約（Guard Rails）は `shared-profile.js` の冒頭コメントを参照。**
フィールドを追加・拡張する前に必ず読む。
