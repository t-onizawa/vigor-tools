## Backlogの構成：Tools と Platform

Backlogは2種類の異なるものを扱う。

```
Tools
  ユーザーの個別の迷いを整理する、単体のツール。
  例：生活防衛資金シミュレーター、サブスク年間総額計算

Platform
  ツール横断で機能する、サイト全体の基盤機能。
  例：判断履歴、マイVIGOR、共通検索
```

Toolsは「1本ずつ公開して反応を見る」対象。
Platformは「サイトの前提を変える」対象なので、より慎重に、
小さい単位で検証しながら進める。

「検索流入・収益化・実装コスト」ではなく、
**「この1本を公開したら、VIGOR TOOLSらしさが一番伝わるか」**で選定。

```

     同じくVISION.md Visionに名指しの「サブスク 年間 いくら」を実現。
     実装が最も軽く、最速で憲法どおりに動けることを証明できる。

     判定・スコア・診断結果を一切出さない、唯一の純粋な整理ツール。
     「自分で決められる状態を作る」を最も純粋な形で体現する。
```

# Platform

サイト全体の基盤機能。個別ツールと違い、一度入れると
全ツールの設計に影響するため、最小単位で検証しながら進める。

| 状態 | 機能名 | 説明 |
|---|---|---|
| 着手中 | 判断履歴 | ツールを使った結果を、ブラウザにだけ自動で残す仕組み。トレンドやグラフは持たず、ツールごとの「最新の結果」だけを記録する。まず生活防衛資金シミュレーター1本で検証する。 |
| 保留 | マイVIGOR | 判断履歴が自然に増えた先に生まれる、状態が見えるホーム画面。判断履歴の価値が検証できるまで着手しない。 |
| 保留 | 共通検索 | ツール数が増えた際の横断検索。目安としてツールが20本を超えたあたりで検討。 |

### 判断履歴 v0.1（検証スコープ）

```
- 対象ツール：生活防衛資金シミュレーターのみ（v0.1では1本に絞る）
- 保存内容：ツールごとの最新の結果1件（時系列やトレンドは持たない）
- 保存先：localStorage（サーバー送信なし、ログイン不要）
- 見せ方：ツール側に「この結果は判断履歴に保存されました。」の一言
  ＋「判断履歴を見る」リンクのみ。専用ホーム画面はまだ作らない。
- 検証すること：履歴が残ること自体が、ユーザーにとって嬉しいか
```

反応が良ければ、独立準備チェックなど2本目を追加し、
その先で初めて「マイVIGOR」の着手を検討する。
