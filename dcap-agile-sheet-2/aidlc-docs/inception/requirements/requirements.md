# Requirements — DCApアジャイルシート v2

**Version**: draft-1  
**Date**: 2026-07-02  
**Phase**: Inception / Requirements Analysis

---

## 背景と目的

既存の dcap-agile-sheet v0.1 を壊さず、別ディレクトリ（dcap-agile-sheet-2）で改善版を開発する。

主な動機は「DCAp の思想をツールの構造として体現すること」。  
現在の v0.1 は入力を整形して表示するだけで、DCAp サイクル（Do→Check→Act→small plan）の「動き」が出力に反映されていない。

---

## 制約（変えないもの）

- HTML / CSS / JavaScript のみ（バックエンドなし、外部APIなし）
- ログインなし
- GitHub Pages で公開できる構成
- スマホ優先
- 3フィールドの入力フォームは維持する

---

## 機能要件

### FR-1: 3フィールド入力フォーム（現状維持）

| フィールド | 内容 | 必須 |
|---|---|---|
| stuck | 今、止まっていること | 任意 |
| goal | 本当は進めたいこと | 任意 |
| next-do | 次の15分でできること | 必須 |

### FR-2: 入力パターンに応じた出力バリエーション

現在の出力は固定文言のみ。  
入力の有無・内容の傾向に応じて、出力文をパターン化して変化させる。

- `stuck` あり / なし で出力文を変える
- `goal` あり / なし で出力文を変える
- `next-do` の長さ・内容傾向でトーンを調整する（拡張余地）
- 「今すぐやるDo」「Check」「Next Do」の3ブロック構造は維持

### FR-3: localStorage による履歴保存

- シート生成のたびに、入力内容と生成日時を localStorage に保存する
- 過去 N 件（上限は設計フェーズで決定）を一覧表示する
- 履歴から過去の内容を参照できる
- 履歴は端末ローカルのみ、サーバー送信なし

### FR-4: DCAp small plan（p）の扱い — TBD

small plan ブロックを出力に追加するかどうかは設計フェーズで決定する。  
選択肢：独立ブロック追加 / 既存「Next Do」をリネーム・強化 / ラベル変更のみ

---

## 非機能要件

- 初回表示が軽い（重い処理なし、外部リソースなし）
- モバイルで操作に迷わない
- 壊れにくいシンプルな実装（過剰な抽象化なし）
